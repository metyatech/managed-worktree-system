# Contributing

## Scope

This repository is the canonical home for the managed-worktree system design and its future implementation.

## Development flow

1. Create an isolated branch for each change.
2. Run `npm install`.
3. Run `npm run verify` before committing.
4. Keep documentation and behavior aligned in the same change set.

## Pull requests

- Keep changes focused.
- Include verification evidence.
- Update the design document when the architecture changes.
